# 데이터 수집 및 전처리 문서

## 1. 데이터 수집 범위
- K-Startup: 예비창업패키지, 초기창업패키지 공고문 PDF
- 중소벤처기업부/소진공: 정책자금 융자 가이드북 PDF
- 지자체(서울/경기 등): 창업지원 공고문 및 관련 조례 PDF

## 2. 전처리 파이프라인
1. PDF 로딩: `PyPDFLoader`
2. 페이지 단위 메타데이터 부착
- `source_file`: 파일명
- `page_number`: 1-based 페이지 번호
3. 텍스트 분할
- `chunk_size=900`, `chunk_overlap=120`
4. 전처리 결과 샘플 저장
- `data/processed/chunks_preview.json`

## 3. 표(Table) 처리 전략
- 1차: PDF 내 텍스트 추출 결과를 chunk에 포함
- 2차(고도화): 표 영역을 Markdown/JSON으로 정규화해 별도 임베딩
- 권장 스키마: `{"table_title": ..., "headers": [...], "rows": [...]}`

## 4. 문서 계층 구조 보존 전략
- 문단/제목 패턴을 정규식으로 감지하여 `section_title` 메타데이터로 저장
- 향후 멀티벡터 리트리버에서 제목/본문 분리 인덱싱 적용

## 5. 불용어 및 도메인 용어
- 일반 불용어 제거는 최소화(정책 문서의 법률 표현 손실 방지)
- 도메인 키워드(예비창업, 업력, 정책자금, 융자, 가점)는 유지

# 발표 자료 초안

## 1. 문제 정의
- 창업지원 공고문은 길고 복잡하여 일반 사용자가 해석하기 어려움
- 잘못된 정보 제공 시 실제 의사결정 리스크가 큼

## 2. 제안 시스템
- 문서 기반 RAG QA 시스템
- 하이브리드 검색(BM25 + Vector)
- 출처(파일명/페이지) 기반 설명 가능한 답변

## 3. 기술 스택
- LangChain
- Chroma(Vector DB)
- OpenAI Embedding + Chat Model
- BM25(rank-bm25)

## 4. 데모 시나리오
- 사용자 프로필: 39세, 서울, 예비 창업자
- 질의: 받을 수 있는 지원금/사업 추천
- 출력: 조건별 요약 + 출처 리스트

## 5. 성능/품질
- 정량: 시나리오 통과율, 출처 명시율
- 정성: 사용자 이해도, 답변 신뢰도

## 6. 한계 및 개선
- 표 구조 정밀 파싱 고도화 필요
- 리랭커 및 메타데이터 필터링 고도화
- 주기적 문서 업데이트 자동화
